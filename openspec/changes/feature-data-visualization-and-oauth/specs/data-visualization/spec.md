## ADDED Requirements

### Requirement: Task Analytics Data Aggregation
The Next.js `/dashboard` engine MUST autonomously interpret the base fetch payload retrieved from `academic_tasks`, calculating the exact mathematical ratios segregating 'pending' status flags from 'completed' status flags effectively formatting a charting payload.

#### Scenario: Array mathematical conversion
- **WHEN** raw Supabase task arrays deploy over the state
- **THEN** the code distills it accurately creating data tuples suitable for visual plotting

### Requirement: Recharts Graphical Output
The dashboard DOM MUST integrate the third-party `recharts` library natively visualizing an explicit graphical Pie/Donut Chart object displaying the previously aggregated completion tuples.

#### Scenario: Visual charting execution
- **WHEN** the UI resolves data
- **THEN** it physically mounts the Recharts `<Pie>` SVG components demonstrating real-time academic progression
