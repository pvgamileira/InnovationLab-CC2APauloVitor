## Why

The goal is to provide students with actionable, AI-driven academic insights. By integrating the Gemini AI model, we can analyze the user's academic performance (subjects and task completion status) to offer strategic recommendations. This acts as a senior academic mentor, helping students understand what to prioritize and how to improve their productivity and academic focus.

## What Changes

- Creation of a new API endpoint at `app/api/gemini-insights/route.js`.
- The endpoint will authenticate the user via Supabase Auth and strictly comply with Row Level Security (RLS) policies using the authenticated `user.id`.
- The endpoint will fetch the user's `subjects` and `academic_tasks` from the Supabase database.
- The endpoint will calculate basic metrics, specifically completed vs pending tasks.
- The data will be sent as JSON context to the Gemini `gemini-1.5-flash` model using the Google Generative AI SDK.
- A custom system prompt will be used: "Aja como um mentor acadêmico sênior. Analise estes dados de desempenho e forneça 3 insights estratégicos curtos (máximo 2 frases cada) sobre o que o aluno deve priorizar."
- The AI insights will be returned as a JSON response to the client.

## Capabilities

### New Capabilities
- `gemini-insights-api`: An API endpoint that provides personalized academic insights generated by Gemini AI based on user data from Supabase.

### Modified Capabilities
- No existing capabilities are modified at the requirements level.

## Impact

- **New Backend Route:** `app/api/gemini-insights/route.js` will be added.
- **Dependencies:** The application must utilize the `@google/generative-ai` SDK and the `@supabase/ssr` (or equivalent Supabase client) configured for the Next.js App Router.
- **Security:** Relies on Supabase Auth and strict RLS for secure, user-specific data retrieval.
